Hi, I'm Fred, the human writing this journal.

If you're reading this, you're probably using AI to help you code. I hope what I share gives you some ideas.

I grew up coding in the 1970s, studied computer science at Swarthmore College in the 1980s where I worked as a sysadmin (although my degree was in political science), then took a detour into law and spent a few years as in-house counsel for an Internet consulting company in the 1990s. All of which to stay it's been a long time waiting for AI to finally get good.

AI is good (it's not a bubble, we really are in a fast take-off world). You can now use AI to code better and faster. Time invested in learning AI for coding pays huge dividends.

So the main reason I'm keeping this journal is to help me think through how I want to use AI, and to record what works best for me.

My daughter is in a PhD computational biology program and is wisely concerned about using AI to do all her coding. She's worried she'll never develop coding skill. That's a risk (although I think it is misplaced) for people who did not grow up having to write assembly code by hand. (I'll write a longer post about that.) For for old farts like me, AI is simply all good. For $20 a month, we get teams of eager interns to boss around for more than 40 hours a week. They'll tackle all the boring parts, leaving the fun stuff for us. We get to decide what is boring and fun: we can tell our AI agents to do whatever it is we don't like doing, and do what we do like doing ourselves. We're the boss.

As much as I'd love to do nothing but code 24/7, I get pulled in a lot of different directions. At my age, once I turn my attention to something else, there's a very short half-life before what I had in my head is gone. This journal is my external memory for coding. Kind of like what the '26 vintage AI models need to do: I'm keeping an external memory between sessions to help me make progress toward long-term goals.

For this journal, I'm using Google's Antigravity for writing and GitHub Pages for hosting. I started by creating a new repo in GitHub, cloned it to my local computer using vscode, then pointed antigravity at the folder and told it in the chat what I wanted to do. I used grill-me (hint: in antigravity / will pull up the menu in the chat so you can select grill-me for planning) to make a plan and then let Gemini do the rest. I'm paying for Google Gemini so I have access to Gemini 3.1 Pro, and I set it to High effort.

For this about file, I'm just editing in antigravity's coding tool. I forget what you call their two different programs, but like cursor I believe antigravity now has a vscode clone and a claude code clone. The vscode clones give you a standard IDE while the claude clode clones give you a chat window with a few doo-dads added on. Having grown up coding, I generally prefer working in the vscode clones. See my posts for more thoughts...
